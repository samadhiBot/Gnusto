# Nitfol

`Nitfol` is the natural language parser used by Gnusto to interpret player commands. It leverages a Core ML model (`Gloth.mlmodelc`) trained specifically on interactive fiction command structures.

## Overview

The core responsibilities of `Nitfol` are:

1.  **Load Model:** Initialize and load the compiled `Gloth.mlmodelc` Core ML word tagger model, which must be present in the module's resources.
2.  **Tag Input:** Utilize the `NaturalLanguage` framework's `NLTagger` to tag individual words (tokens) in the player's input string according to a custom tag scheme (`Nitfol`).
3.  **Structure Command:** Interpret the tagged tokens to identify the key components of the command: verb, direct object (and its modifiers), preposition, and indirect object (and its modifiers).
4.  **Return Structured Data:** Provide the parsed components in a standardized `Command` struct.

## Dependencies

- `CoreML`: Required indirectly for loading the `.mlmodelc` file.
- `NaturalLanguage`: Provides the `NLTagger` and related infrastructure for applying the Core ML model to text.

## Core ML Model (`Gloth.mlmodelc`)

Nitfol relies entirely on the `Gloth.mlmodelc` file generated by the [Gloth](Gloth-README.md) executable. This model has been trained to recognize the following tags within player input:

- `verb`: The main action word.
- `directObject`: The primary noun receiving the action.
- `preposition`: Words like "in", "on", "with", "to", etc.
- `determiner`: Articles like "a", "an", "the" (these are ignored by the parser).
- `modifier`: Adjectives or adverbs describing an object.
- `indirectObject`: The secondary noun, often linked by a preposition.

**Important:** The `Gloth.mlmodelc` file **must** be compiled and placed within the `Sources/Nitfol/Resources/` directory for Nitfol to initialize successfully.

## Usage

Using Nitfol involves initializing it and then passing raw command strings to its `parse` method.

```swift
import Nitfol

do {
    // 1. Initialize Nitfol (loads the Gloth.mlmodelc)
    let parser = try Nitfol()

    // 2. Parse player input
    let inputString = "take the rusty key with the small tag"
    let command = parser.parse(inputString)

    // 3. Access the parsed components
    print(command.verb) // Optional("take")
    print(command.directObject) // Optional("key")
    print(command.directObjectModifiers) // ["rusty"]
    print(command.preposition) // Optional("with")
    print(command.indirectObject) // Optional("tag")
    print(command.indirectObjectModifiers) // ["small"]

} catch let error {
    print("Failed to initialize Nitfol: \(error)")
    // Handle the error, likely indicating the model file is missing.
}
```

## Parsing Logic

Nitfol uses the `NLTagger` to enumerate tags for each word. It then applies some basic heuristics:

- Determiners (`the`, `a`, `an`) are ignored.
- The first word tagged as `directObject` becomes the `command.directObject`.
- Any subsequent word tagged as `directObject` is assumed to be the `command.indirectObject` _unless_ a `preposition` and/or `indirectObject` tag is explicitly found later.
- Words tagged as `modifier` occurring _before_ the `directObject` is identified are added to `command.directObjectModifiers`.
- Words tagged as `modifier` occurring _after_ the `directObject` is identified are added to `command.indirectObjectModifiers`.

This logic provides a reasonable interpretation for common command patterns like `[verb] [modifiers] [direct_object] [preposition] [modifiers] [indirect_object]`, but may not cover all grammatical edge cases.

## Output (`Command` Struct)

The `parse(_:)` method returns a `Command` struct (defined in `Sources/Nitfol/Command.swift`) with the following optional properties:

- `verb: String?`
- `directObject: String?`
- `directObjectModifiers: [String]`
- `preposition: String?`
- `indirectObject: String?`
- `indirectObjectModifiers: [String]`

If a component isn't found in the input string or identified by the tagger, the corresponding property will be `nil` (or an empty array for modifiers).
